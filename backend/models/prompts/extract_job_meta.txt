Extract structured metadata from a job description. Output valid minified JSON with exactly these keys: {"company":"...","role_title":"...","top_requirements":["...","...","..."],"why_company_fact":"...","jd_domain_tags":["...","...","..."],"keywords_exact":["..."]}

- company: the verbatim company name as written in the job posting. Use "" (empty string) if the posting does not clearly name the company. NEVER invent a company name. NEVER output generic placeholders like "Tech Innovators", "Tech Innovators Inc.", "the company", "a leading company", "Example Corp", "Acme", "Company X", or any similar generic/example placeholder.
- role_title: the verbatim job title from the posting (e.g. "Software Engineer I, Backend"). Use "" (empty string) if not explicitly stated. NEVER invent a title. NEVER output "Senior Backend Developer" or any generic title not found in the posting.
- top_requirements: the 3 most important skills/experiences the posting emphasizes.
- why_company_fact: one short phrase describing something specific about the company grounded in the posting (e.g. "mission to democratize finance", "trains frontier language models"). If the posting does not state anything specific and verifiable, use "" (empty string). NEVER invent awards, rankings, or cultural claims.
- jd_domain_tags: 3-5 short lowercase tags describing the job domain, such as backend, distributed, fintech, fullstack, ai, data, ml, platform.
- keywords_exact: max 20 verbatim technology names, frameworks, tools, databases, and concepts from the posting.

Output ONLY the JSON. No code fence. No prose.
